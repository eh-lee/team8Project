# Commit Rule

    - [feat]
      - 새로운 기능 추가
    - [fix]
      - 수정
    - [chore]
      - 빌드 업무 수정, 패키지 매니저 수정, 의존성 설치 등

    - [test]
      - 테스트 코드, 미완인 코드 커밋 시

    - [remove]
      - 기능과 관련된 삭제

    - [design]
      - CSS 전반
      - 디자인 추가, 수정, 삭제 등 전부

    - [style]
      - 공백, 문장 부호 제거-추가, 코드 포맷 변경 등 (내용 수정이 없는 경우)

    - [refactor]
      - 리팩토링, 각주 제거, 지저분한 거 지울 때도..

    - [docs] ~ README.md


    - 커밋 머시기가 중복될 경우, 중요한 것을 헤드에 적기. 헷갈리는 경우 상의하기.
